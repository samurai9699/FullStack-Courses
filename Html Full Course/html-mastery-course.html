Module 01 — Foundations

How HTML Works

Understand what HTML actually is, what it's not, and how the browser turns text into a webpage.

What is HTML?

HTML (HyperText Markup Language) is the skeleton of every web page. It describes the structure and meaning of content — not how it looks (that's CSS) and not how it behaves (that's JavaScript).

Think of a webpage like a house: HTML is the walls, floors, and rooms. CSS is the paint and furniture. JavaScript is the electricity and plumbing.

💡
Mental Model HTML is a set of instructions you write for the browser. The browser reads them and renders visual content on screen. You're describing content, not drawing it.

How the Browser Renders HTML

When you open a webpage, the browser does this:

  1. Fetches the HTML file from a server (or disk)
  2. Parses the HTML — reads it top to bottom, building a DOM tree
  3. Fetches linked CSS, JavaScript, images
  4. Renders — calculates layout and paints pixels on screen
✅
The DOM DOM = Document Object Model. It's a tree of all the HTML elements on a page. JavaScript interacts with the DOM to change what you see. HTML creates the initial DOM.

Anatomy of an HTML Element

Every HTML element follows the same pattern:

<!-- Opening tag + Content + Closing tag -->
<p class="intro">Hello, world!</p>

<!-- Self-closing (void) element — no closing tag -->
<img src="logo.png" alt="Logo" />

Attributeclass="intro"Extra info about the element
ContentHello, world!What goes inside
Closing tag</p>Ends the element
Void element<img />No closing tag needed

Block vs Inline Elements

This is one of the most important mental models in HTML. Every element is either block or inline by default.

TypeBehaviorExamples
BlockTakes full width, starts on a new linediv, p, h1–h6, ul, li, section
InlineSits within a line of text, only as wide as contentspan, a, strong, em, img, code
⚠️
CSS overrides this CSS can change any element's display behavior. But semantically, block and inline still matter for accessibility and screen readers.

Interview Prep Module 1 Questions
What's the difference between HTML, CSS, and JavaScript?
HTML defines structure/content, CSS defines presentation/styling, JavaScript defines behavior and interactivity. They have a separation of concerns by design — HTML should work without CSS or JS (progressive enhancement).
What is the DOM and how does it relate to HTML?
HTML is the source text. When the browser parses HTML, it creates the DOM — an in-memory tree of objects. JavaScript manipulates the DOM (not the raw HTML text). Changes to the DOM update what's rendered, but don't change the original HTML file.
What's the difference between block and inline elements?
Block elements occupy the full width available and start on a new line (e.g. div, p, h1). Inline elements flow within text and are only as wide as their content (e.g. span, a, strong). CSS display can change this behavior.
Module 02 — Foundations

Document Structure

The skeleton every HTML file must have — and exactly what each part does.

The Minimal Valid HTML5 Document

<!-- This is a valid, complete HTML5 page -->
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
  </body>
</html>

Line-by-line breakdown

LineWhat it does
<!DOCTYPE html>Not a tag — it's a declaration. Tells the browser: "use HTML5 mode". Always first line.
<html lang="en">Root element. lang is critical for accessibility and SEO — tells screen readers and search engines the document's language.
<head>Container for metadata — none of it renders visibly on the page.
<meta charset="UTF-8">Sets character encoding. UTF-8 supports every language including emoji. Must be in the first 1024 bytes.
<meta name="viewport">Makes the page responsive on mobile. Without this, mobile browsers zoom out and your site looks broken.
<title>What appears in browser tabs and search results. SEO-critical. Max ~60 chars.
<body>Everything here is rendered visibly on screen.
🚫
Never skip these Missing charset can cause garbled text. Missing viewport breaks mobile layout. Missing lang fails accessibility audits. All three are required in every real project.

The <head> Element Deep Dive

The head is invisible but powerful. It's where you load CSS, define SEO, set icons, and configure the page.

<head>
  <!-- Encoding -- MUST be first -->
  <meta charset="UTF-8" />

  <!-- Responsive viewport -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  <!-- Page title (shown in browser tab + search results) -->
  <title>Product Page | My Brand</title>

  <!-- External CSS file -->
  <link rel="stylesheet" href="styles.css" />

  <!-- Favicon -->
  <link rel="icon" href="favicon.ico" />
</head>

Nesting Rules

HTML elements nest inside each other, forming a tree. The rules:

  • Correct: child elements must be fully inside the parent
  • Correct: block elements can contain inline elements
  • Wrong: inline elements generally shouldn't contain block elements
  • Wrong: overlapping tags (opening one before closing another from a sibling)
<!-- ✅ Correct nesting -->
<div>
  <p>This is a <strong>paragraph</strong> with bold text.</p>
</div>

<!-- ❌ Wrong — overlapping tags -->
<p><strong>Don't do </p>this</strong>

<!-- ❌ Wrong — inline wrapping block -->
<span><div>Bad idea</div></span>

Interview PrepModule 2 Questions
What goes in the <head> vs <body>?
Head contains metadata — title, charset, viewport, CSS links, JS scripts with defer, SEO meta tags, and favicons. Nothing in the head renders visibly. Body contains all visible page content — text, images, forms, etc.
Why does the charset meta tag need to be first?
The browser starts parsing HTML immediately. If it encounters text before knowing the charset, it may decode characters incorrectly. Placing charset first (within the first 1024 bytes) ensures the browser knows encoding before it encounters any text content.

Text & Typography

All the elements for marking up text — headings, paragraphs, emphasis, quotes, and more.

Headings — h1 through h6

Headings establish a document hierarchy. They are critical for SEO, accessibility, and screen reader navigation.

<h1>Page Title — only one per page</h1>
<h2>Major Section</h2>
<h3>Subsection</h3>
<h4>Sub-subsection</h4>
<h5>Rarely used</h5>
<h6>Almost never used</h6>
🚫
Don't use headings for size — that's CSS's job Never use <h3> because "I want smaller text". Use headings only to express document hierarchy. Resize them with CSS. Screen readers build a navigation tree from headings — wrong use breaks accessibility.

Paragraphs & Line Breaks

<!-- A paragraph -->

<!-- Hard line break within text (use sparingly) -->

</p>

<!-- Horizontal rule — thematic break -->
<hr>
⚠️
Don't abuse <br> Using multiple <br> tags for spacing is a bad habit. Use CSS margin and padding instead.

The <span> and <div> Elements

These are the generic containers — they have no semantic meaning. Use them only when no other element fits.

<!-- div = block container -->
<div class="card">
  <p>Content inside a card layout</p>
</div>

<!-- span = inline container -->
<p>Price: <span class="price">$29.99</span> (limited time)</p>
✅
div soup is a code smell Overusing <div> everywhere is a beginner mistake. Prefer semantic elements like <article>, <section>, <nav> (covered in Module 8). Use <div> only for layout grouping when nothing semantic fits.

Interview PrepModule 3 Questions
Why use <strong> instead of CSS bold? Why does it matter?
Screen readers announce <strong> with added emphasis — it has semantic meaning. Using CSS font-weight: bold on a <span> produces the same visual result but zero semantic meaning. Accessible HTML uses the right elements for the right reasons.
When would you use <time datetime="...">?
When displaying dates or times that should be machine-readable — blog post timestamps, event dates, opening hours. The datetime attribute provides the ISO 8601 format for search engines and assistive tech, while the element content can be in any human-readable format.

Links & Navigation

Anchors are the foundation of the web. Master every URL type, target, and accessibility pattern.

The Anchor Element

<!-- External link -->
<a href="https://github.com">Visit GitHub</a>

<!-- Open in new tab (always add rel for security) -->
<a href="https://docs.mozilla.org" target="_blank" rel="noopener noreferrer">
  MDN Docs
</a>

<!-- Internal link (same site) -->
<a href="/about">About Us</a>

<!-- Relative link -->
<a href="../contact.html">Contact</a>

<!-- Anchor link (jump within page) -->
<a href="#pricing">Go to Pricing</a>
<section id="pricing">...</section>

<!-- Email link -->
<a href="mailto:hello@example.com?subject=Hi&body=Hello!">Email Us</a>

<!-- Phone link (mobile) -->
<a href="tel:+254700000000">Call Us</a>

<!-- File download -->
<a href="/report.pdf" download="Annual-Report.pdf">Download Report</a>
🚫
target="_blank" Security Risk When you open a link in a new tab without rel="noopener noreferrer", the new page can access window.opener and potentially redirect your page (tab-napping attack). Always pair target="_blank" with rel="noopener noreferrer".

Navigation Landmark

<!-- Site navigation -- use <nav> for semantic grouping -->
<nav aria-label="Main">
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li><a href="/blog" aria-current="page">Blog</a></li>
  </ul>
</nav>
✅
aria-current="page" Add this to the active navigation link. Screen readers announce it as "current page", and you can style it with CSS using [aria-current="page"].

Interview PrepModule 5 Questions
What is the security risk with target="_blank"?
Without rel="noopener", the opened tab gets a reference to the opener window via window.opener. A malicious site can use this to redirect the original tab to a phishing page. noopener breaks this reference. noreferrer additionally hides the referrer header.
What's the difference between absolute, relative, and root-relative URLs?
Absolute: full URL including protocol (https://example.com/page). Relative: path relative to current file (../images/logo.png). Root-relative: path from domain root (/images/logo.png) — portable across directories, preferred for internal links.

Images & Media

Images, video, audio, iframes — and the performance and accessibility patterns that go with each.

Images

<!-- Basic image -->
<img src="hero.jpg" alt="Developer working at desk" width="800" height="600" />

<!-- Lazy-loaded image (offscreen images won't block page load) -->
<img src="below-fold.jpg" alt="..." loading="lazy" />

<!-- Priority image (LCP image -- don't lazy load this) -->
<img src="hero.jpg" alt="..." fetchpriority="high" />

<!-- Decorative image (screen readers skip it) -->
<img src="divider.png" alt="" role="presentation" />
🚫
alt="" is not optional Every <img> needs an alt attribute. Missing alt is a WCAG failure and breaks screen readers. For decorative images, use alt="" (empty). For informational images, describe the image's content and function.

Responsive Images with srcset

<!-- Browser picks the right resolution -->
<img
  src="hero-800.jpg"
  srcset="hero-400.jpg 400w, hero-800.jpg 800w, hero-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw, 800px"
  alt="Hero image"
/>

Picture Element — Art Direction

<picture>
  <!-- Modern format first, fallback last -->
  <source srcset="hero.avif" type="image/avif" />
  <source srcset="hero.webp" type="image/webp" />
  <img src="hero.jpg" alt="Hero image" />
</picture>

Figure & Figcaption

<figure>
  <img src="chart.png" alt="Bar chart showing Q3 revenue by region" />
  <figcaption>Figure 1: Q3 Revenue by Region (2024)</figcaption>
</figure>

Video

<video controls width="800" poster="thumbnail.jpg" preload="metadata">
  <source src="video.webm" type="video/webm" />
  <source src="video.mp4" type="video/mp4" />
  <!-- Captions for accessibility -->
  <track kind="captions" src="captions.vtt" srclang="en" label="English" default />
  <p>Your browser doesn't support video. <a href="video.mp4">Download</a></p>
</video>

<!-- Background/decorative video -->
<video autoplay muted loop playsinline aria-hidden="true">
  <source src="bg.mp4" type="video/mp4" />
</video>

Iframes

<!-- Embedding YouTube -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  title="Video title"
  width="560" height="315"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope"
  allowfullscreen
  sandbox="allow-scripts allow-same-origin"
  loading="lazy"
></iframe>
⚠️
Always set title on iframes Screen readers need the title attribute to describe what's in the iframe. An untitled iframe gets announced as a meaningless frame.

    }

</script>

Interview PrepModule 7 Questions
What's the difference between GET and POST in a form?
GET appends form data to the URL as query parameters — visible, bookmarkable, limited in size. Use for searches. POST sends data in the request body — not visible in URL, no size limit, more secure for sensitive data. Use for login, registration, payments, any data mutation.
What is a CSRF token and why is it in a hidden input?
Cross-Site Request Forgery token is a server-generated secret embedded in the form. When submitted, the server validates it matches the session token. This prevents malicious sites from submitting forms on behalf of logged-in users. Hidden inputs are invisible to users but submitted with form data.

Advanced DOM Manipulation & Event Delegation

<script>
  // DOMContentLoaded fires when HTML is parsed (CSS/images might still be loading)
  // load fires when EVERYTHING is fully loaded
  document.addEventListener('DOMContentLoaded', () => {
    console.log('DOM is ready!');
  });

  // Event Delegation: Listen on parent, catch events from children
  // Extremely performant for lists/tables with many items
  document.querySelector('.list').addEventListener('click', (e) => {
    // closest() travels up the DOM to find a matching selector
    const btn = e.target.closest('.delete-item');
    if (btn) {
      deleteItem(btn.dataset.id);
    }
  });

  // insertAdjacentHTML: Safely and efficiently insert HTML without destroying existing elements
  const container = document.querySelector('.container');
  container.insertAdjacentHTML('beforeend', '<div>New Content</div>');
</script>

Interview PrepMaster-Level Review — All Topics
What is the difference between defer and async on a script tag?
defer downloads in parallel and executes after HTML parsing is complete, maintaining document order. async downloads in parallel and executes immediately when ready, in any order. Use defer for application scripts, async for independent third-party scripts like analytics.
What is semantic HTML and why does it matter?
Semantic HTML uses elements that carry meaning about their content — <article>, <nav>, <time> — rather than generic containers like <div>. It matters because: (1) screen readers use it to build navigation, (2) search engines use it to understand content, (3) it makes code maintainable.
What is WCAG and what are the three levels of compliance?
WCAG (Web Content Accessibility Guidelines) is the international standard for web accessibility. Levels: A (minimum), AA (standard target for most sites — legally required in EU/US contexts), AAA (highest, often impractical for all content). AA compliance covers most accessibility needs including color contrast, keyboard navigation, and screen reader support.
Explain the difference between GET and POST for form submission.
GET appends form data to URL as query string — visible, bookmarkable, idempotent. Limited to ~2000 chars. POST sends data in request body — not visible in URL, no size limit, not idempotent. Use GET for search forms. Use POST for any data that creates, updates, or deletes — login, checkout, file upload.
What causes layout shift (CLS) and how do you fix it in HTML?
CLS happens when elements move after initial render — typically caused by images loading without reserved space. Fix: always specify width and height attributes on <img> so the browser allocates space before the image loads. Also caused by dynamically injected content (ads, embeds) that pushes content down.
What is the difference between <input type="submit"> and <button type="submit">?
<button> can contain HTML (icons, complex content). <input type="submit"> can only have a plain text value. Both submit the form. In modern HTML, <button type="submit"> is preferred because it's more flexible and stylable. Always explicitly set type — a <button> without type defaults to submit which can accidentally submit forms.
How do you make a custom interactive component (e.g., a modal) accessible?
Use role="dialog", aria-modal="true", and aria-labelledby pointing to the modal's heading. When opened: move focus inside using focus(). Trap focus within the modal (Tab should cycle inside). When closed: return focus to the trigger button. Add Escape key handler. Use the native <dialog> element which handles most of this automatically.
What is the Content Security Policy and how does it relate to HTML?
CSP is a security header (set by server or <meta http-equiv="Content-Security-Policy">) that restricts which sources scripts, styles, and images can load from. It blocks XSS by preventing injected inline scripts from executing. Tightly scoped CSP is a major defense layer — e.g., script-src 'self' allows only scripts from your own domain.

Module 15 — Advanced

Web Components

Build reusable, encapsulated HTML tags without a framework. The future of the web platform.

The 3 Pillars of Web Components

  • Custom Elements: JavaScript APIs for defining new HTML tags.
  • Shadow DOM: Encapsulated CSS and DOM that doesn't leak out or get overridden.
  • HTML Templates: Reusable chunks of markup (<template> and <slot>).

A Complete Web Component

<!-- Usage in HTML -->
<user-profile name="Jane Doe">
  <span slot="role">Admin</span>
</user-profile>

<script>
  class UserProfile extends HTMLElement {
    constructor() {
      super();
      // Attach Shadow DOM
      this.attachShadow({ mode: 'open' });
    }

    // Lifecycle hook: added to DOM
    connectedCallback() {
      const name = this.getAttribute('name') || 'User';
      this.shadowRoot.innerHTML = `
        <style>
          .card { border: 1px solid #ccc; padding: 10px; border-radius: 8px; }
          h3 { margin: 0; color: blue; } /* Won't affect h3 tags outside! */
        </style>
        <div class="card">
          <h3>${name}</h3>
          <p>Role: <slot name="role">Guest</slot></p>
        </div>
      `;
    }
  }

  // Register the custom element (must contain a hyphen)
  customElements.define('user-profile', UserProfile);
</script>

Module 16 — Advanced

HTML Validation & Debugging

How to guarantee your HTML is robust, accessible, and correctly parsed using professional tools.

The Forgiving Nature of HTML

Browsers try incredibly hard to fix broken HTML. If you forget a closing tag, the browser guesses where it should go. This is dangerous. It leads to inconsistent rendering across different browsers and silent accessibility failures.

The W3C Validator

Always run your final pages through the W3C Markup Validation Service.

  • Checks for missing required attributes (like alt on images)
  • Catches improper nesting (e.g., a block element inside an inline <span>)
  • Identifies deprecated tags

Debugging with DevTools

Use your browser's Developer Tools (F12 or Right Click -> Inspect) to view the Live DOM. The Live DOM is what the browser actually rendered, which might be different from your source code if you had errors or JS modifications.

  • Elements Tab: Inspect the DOM tree, modify classes on the fly.
  • Accessibility Tree: See exactly what a screen reader sees. Elements with aria-hidden="true" disappear here.
