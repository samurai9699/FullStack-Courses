# Dev Reference Library

A collection of self-contained developer courses I built as offline-first HTML references. No login. No account. No dependencies. Download a file, open it in a browser, and work through it anywhere.

Each course is a single HTML file covering a topic from first principles through to practical application. The goal was to produce resources that are dense, opinionated, and actually usable without an internet connection.

---

## Courses

| Topic | What it covers |
|---|---|
| HTML | Document structure, semantics, accessibility fundamentals |
| CSS | Layout systems, Flexbox, Grid, responsive design |
| Tailwind CSS | Utility-first workflow, custom config, component patterns |
| JavaScript | Core language, DOM, async patterns, modern ES features |
| Git / CI-CD | Version control workflows, branching strategies, pipelines |
| PostgreSQL | Relational modelling, queries, indexing, transactions |
| NestJS | Modular architecture, decorators, providers, REST APIs |
| WebSockets | Real-time communication, events, connection lifecycle |
| Vite | Build tooling, dev server, plugin system, bundling |
| Docker | Containers, images, Compose, multi-stage builds |
| AWS | Core services, IAM, compute, storage, networking basics |
| System Design | Scalability patterns, databases, caching, trade-off thinking |
| Command Line | Shell navigation, scripting, process management |
| Technical Writing | Documentation structure, API docs, developer-facing content |
| Prompt Engineering | LLM fundamentals, prompt patterns, structured outputs |

---

## Why offline-first

Most developer learning resources require an active connection, an account, or a runtime environment. These files require none of that. The entire content of each course lives inside a single HTML file. Useful for slow connections, travel, or environments where you cannot guarantee access.

---

## Usage

Clone the repo or download individual files.

```bash
git clone https://github.com/samurai9699/FullStack-Courses.git
```

Open any `.html` file directly in your browser. No build step. No install.

---

## Stack

Pure HTML and CSS. Intentionally no JavaScript frameworks or build tools, since the point is zero-dependency portability.
